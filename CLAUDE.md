# CLAUDE.md — PC 업그레이드 진단 사이트

이 파일은 Claude Code가 이 프로젝트에서 작업할 때 매 세션 시작 시 읽는 지침서다.
사용자를 도울 때는 **"클대리"** 라는 호칭을 사용해도 좋다.

---

## 1. 프로젝트 한 줄 정의

사용자가 **현재 쓰는 PC 사양(CPU/GPU/RAM)** 을 입력하면 → 병목을 진단하고 → 어느 부품을
바꾸면 체감 효과가 가장 큰지 + **그 부품의 실제 가격·구매처**까지 보여주는 정적 사이트.
수익은 애드센스가 아니라 **제휴 커머스 수수료(쿠팡파트너스·다나와)** 중심.

### 차별점 (이게 핵심이다 — 단순 병목 계산기 클론이 되면 안 됨)
1. **비용 대비 효과**: "병목 12%"가 아니라 "X만원짜리 Y로 바꾸면 체감 성능 이만큼 ↑"
2. **게임 단위 체감**: "fps 65→110"보다 "안 되던 사이버펑크 이제 원활하게 실행"
3. **정직한 진단**: 경쟁 도구가 "단순 비율이라 부정확"하다고 비판받는 점을 한계 명시로 역이용

---

## 2. 기술 스택 / 아키텍처 원칙

- **정적 사이트 우선**: 진단 로직은 100% 클라이언트 JS로 동작. 서버 없이 빌드·배포 가능해야 한다.
- 호스팅 후보: Vercel / Netlify / Cloudflare Pages (아직 미정 — 사용자가 계정 생성 예정)
- **가격 데이터만** 외부 API(다나와/네이버 쇼핑)가 필요하고, 나머지는 전부 정적.
- API 키는 **절대 클라이언트 코드에 노출하지 않는다.** 브라우저 직접 호출은 키 노출 + CORS 문제.
  - 가격 연동은 아래 3단계 중 택1 (초기엔 방식 A 권장):
    - **A. 대략 가격대 표시** — 사이트엔 "30만원대"만, 정확한 현재가는 제휴 링크 클릭 후 확인. (서버 불필요)
    - **B. 주기적 재빌드** — 빌드 시 가격을 구워넣고 GitHub Actions로 매일 재빌드. 호스팅은 정적 유지.
    - **C. 서버리스 함수** — Vercel/Cloudflare 함수에 키 숨기고 6~12h 캐싱.

> 지금 단계에서는 **API·제휴 가입이 안 된 상태**다. 따라서 가격/구매 링크는 **임시 mock 데이터**로
> 구현하고, 실제 연동 지점은 함수로 분리해서 나중에 교체만 하면 되게 둔다.

## 3. 데이터 설계 (CSV 2종 + mock)

데이터는 사용자가 직접 구축 중. 지금은 **임시 샘플 데이터(게임 8~10개, 부품 10개씩)** 로 전체
흐름을 먼저 검증한 뒤, 나중에 실제 100개 게임 / 150개 부품으로 확장한다.

### 3-1. GPU 점수표 `data/gpus.csv`
| 컬럼 | 설명 |
|---|---|
| `model` | 모델명 (표기 규칙 통일 필수, 아래 참고) |
| `brand` | NVIDIA / AMD / Intel |
| `score` | PassMark 기반 단일 점수 |
| `year` | 출시연도 |
| `psu_watt` | 권장 PSU 용량(W) — 파워 호환 체크용 |

### 3-2. CPU 점수표 `data/cpus.csv`
| 컬럼 | 설명 |
|---|---|
| `model` | 모델명 |
| `brand` | Intel / AMD |
| `score` | PassMark 기반 단일 점수 |
| `cores` / `threads` | 코어·스레드 수 |
| `year` | 출시연도 |
| `socket` | 소켓 타입 — 메인보드 호환 체크용 |

### 3-3. 게임 사양표 `data/games.csv`
| 컬럼 | 예시 |
|---|---|
| `name` | 사이버펑크 2077 |
| `min_cpu` / `min_gpu` / `min_ram` | i5-3570K / GTX 780 / 12 |
| `rec_cpu` / `rec_gpu` / `rec_ram` | i7-12700 / RTX 2060 SUPER / 16 |

### ⚠️ 모델명 표기 규칙 (어기면 매칭이 깨진다)
- 점수표 · 게임표 · API 검색어 **세 곳에서 모델명 표기를 반드시 동일하게** 쓴다.
- 규칙: **영문 모델명 그대로** (예: `RTX 4060` — "지포스" 같은 접두어 금지).
- 게임 사양표엔 구형·보급형 부품이 많이 등장 → 점수표는 "최신 인기 부품" +
  "게임표에 등장하는 모든 모델"을 합쳐서 구성해야 매칭이 끊기지 않는다.

## 4. 진단 로직 명세

### 4-1. 병목 진단
- CPU 점수와 GPU 점수의 균형을 보고 어느 쪽이 발목을 잡는지 판정.
- 기존 도구들이 RAM/해상도/오버클럭을 무시해서 부정확하다는 비판을 받음 →
  최소한 **모니터 해상도**는 입력받아 반영하고, 반영 못 하는 요소는 결과에 **한계를 명시**한다.

### 4-2. 게임별 실행 등급 (핵심 기능)
- 각 게임의 min/rec GPU(및 CPU)를 같은 점수 척도로 환산해 사용자 점수와 비교.
- 3단계 판정:
  - 사용자 점수 < 최소 → **실행 어려움**
  - 최소 ≤ 사용자 점수 < 권장 → **버벅임 가능**
  - 권장 ≤ 사용자 점수 → **원활하게 실행**
- 추천 부품으로 교체했을 때 등급이 오르는 게임만 추려 **"전: 실행 어려움 → 후: 원활"** 로 보여준다.

### 4-3. 추천 + 비용 대비 효과
- 약점 부품을 한 단계 윗급으로 추천 → 추천 모델명 도출.
- 호환성 체크: 추천 CPU의 `socket`이 기존과 다르면 "메인보드도 교체 필요" 경고.
  추천 GPU의 `psu_watt`이 기존 파워를 초과하면 "파워 교체 필요" 경고.
- 추천 모델명을 가격 조회 함수(현재는 mock)에 넘겨 가격대 + 구매 링크 표시.

> ⚠️ 게임사 공개 사양은 보통 **1080p 기준**이다. 1440p/4K 사용자는 체감이 다르므로
> 결과에 "1080p 기준" 안내를 반드시 노출한다. 과장 금지 — 신뢰가 이 사이트의 자산이다.

## 5. 사용자 흐름 (UI)

1. **입력** — CPU·GPU·RAM·주로 하는 게임·모니터 해상도를 **검색형 드롭다운**(타이핑 자동완성)으로 선택.
   - 브라우저는 실제 하드웨어를 자동으로 못 읽음 → 사용자가 직접 선택(업계 표준).
   - GPU만 WebGL로 "추정값" 자동 채움 정도는 보조로 가능(정확도 100% 아님 → 확인 요청).
2. **진단** — 클라이언트 JS로 병목 계산 + 게임 등급 판정.
3. **결과: 성능 비교 그래프** — 게임별 현재 vs 추천 사양 예상 fps, 등급 변화(전/후) 시각화.
4. **추천 부품 매칭** — 추천 모델명 → 가격 조회(mock) → 상품·가격대 표시.
5. **구매 링크** — 쿠팡파트너스·다나와 제휴 링크(현재는 placeholder URL).

## 6. 권장 빌드 순서 (이 순서로 진행)

1. 프로젝트 스캐폴딩 + `data/` 에 임시 샘플 CSV(게임 8~10, 부품 10씩) 생성.
2. CSV 로더 + 진단 엔진(병목, 게임 등급) 순수 함수로 구현 + 단위 테스트.
3. 입력 폼(검색 드롭다운) → 진단 호출 → 결과 화면 연결.
4. 성능 비교 그래프 + 게임 전/후 등급 카드 UI.
5. 가격 조회 함수를 **mock으로** 구현(실제 API 자리만 분리) + 구매 링크 placeholder.
6. 호환성 경고(소켓/PSU) 로직 추가.
7. 반응형·접근성·"1080p 기준"·진단 한계 안내 등 신뢰 요소 마감.

> 작은 검증 세트로 1~4를 먼저 끝까지 돌려본 뒤 실제 데이터로 확장한다. 한 번에 100개 채우지 말 것.
